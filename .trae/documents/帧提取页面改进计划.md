# 帧提取页面改进计划

## 研究结论

通过分析代码库，我发现了以下需要解决的问题：

1. **拖动功能问题**：
   - 帧在手动拖动时没有边界限制，可能会超出放置区域
   - 没有碰撞检测，帧可能会重叠
   - 拖动时没有吸附到网格的功能

2. **间距问题**：
   - 间距设置只在自动排列时生效
   - 手动拖动和导出时没有正确应用间距
   - 没有视觉反馈显示间距

3. **导出功能问题**：
   - 导出前没有预览功能
   - 用户无法在导出前检查最终效果
   - 缺少确认步骤

## 文件和模块需要修改

1. `index.html`：
   - 添加新的导出预览模态框
   - 修改导出流程

2. `js/frame-extract.js`：
   - 修改 `handleExtractTargetMove` 函数，添加边界限制和碰撞检测
   - 修改 `autoArrangeExtractTarget` 函数，确保间距正确应用
   - 修改 `exportAsSpriteSheet` 函数，正确应用间距
   - 添加导出预览和确认功能
   - 添加拖动时的网格吸附功能

## 修改步骤

### 1. 改进拖动功能
- 在 `handleExtractTargetMove` 函数中添加边界检测，确保帧不会超出放置区域
- 添加简单的碰撞检测，避免帧重叠
- 添加网格吸附功能，让帧在拖动时自动对齐到网格

### 2. 确保间距正确应用
- 修改 `autoArrangeExtractTarget` 函数，确保间距设置被正确计算和应用
- 在拖动时提供间距视觉反馈
- 修改 `exportAsSpriteSheet` 函数，确保导出时正确应用间距

### 3. 添加导出预览和确认功能
- 创建新的模态框用于预览导出效果
- 修改导出流程，先显示预览，用户确认后再导出
- 在预览模态框中显示最终的精灵表效果

## 潜在依赖和注意事项

1. 需要确保所有修改与现有代码兼容
2. 需要考虑性能，特别是碰撞检测和预览渲染
3. 需要保持用户体验的连贯性
4. 需要确保预览在不同浏览器和设备上都能正常工作

## 风险处理

1. **性能风险**：添加碰撞检测可能会影响性能，特别是在帧数量较多时
   - 解决方案：实现优化的碰撞检测算法，只检测最近的帧

2. **兼容性风险**：新添加的功能可能与某些浏览器不兼容
   - 解决方案：使用标准的 Web API，避免使用实验性功能

3. **用户体验风险**：改动可能会影响现有用户的工作流程
   - 解决方案：保持现有功能不变，只添加新功能和改进现有功能

## 预期效果

1. 用户可以流畅地拖动帧，不必担心帧超出区域或重叠
2. 间距设置在所有场景下都能正确生效
3. 用户在导出前可以预览最终效果，避免出错
4. 整体用户体验得到提升

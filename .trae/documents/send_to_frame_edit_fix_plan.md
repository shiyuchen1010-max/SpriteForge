# 发送到帧编辑功能修复计划

## 仓库研究结论

经过分析SpriteForge项目的代码，我发现：

1. 发送到帧编辑功能的实现位于 [frame-extract.js](file:///workspace/sprite-forge-user/js/frame-extract.js) 文件中的 `sendToFrameEdit` 函数
2. 该函数在用户选择"发送到帧编辑"选项并点击确定后被调用
3. 函数的实现逻辑是：
   - 将提取的帧转换为精灵编辑页面可识别的格式
   - 创建一个包含所有帧的canvas
   - 将canvas转换为data URL
   - 加载这个data URL作为新的图片
   - 跳转到精灵编辑页面并显示这些帧

## 问题分析

通过检查代码，我发现问题出在 `sendToFrameEdit` 函数中：

1. 当创建新的Image对象并设置onload回调后，没有设置 `img.src = dataUrl`，导致图片不会加载
2. 由于图片不会加载，onload回调永远不会被触发，因此页面不会跳转到精灵编辑页面

## 需要修改的文件

1. [frame-extract.js](file:///workspace/sprite-forge-user/js/frame-extract.js) - 修复 `sendToFrameEdit` 函数

## 修改步骤

### 1. 修复 sendToFrameEdit 函数
- 在创建Image对象并设置onload回调后，添加 `img.src = dataUrl` 语句，确保图片能够加载
- 这样当图片加载完成后，就会触发onload回调，从而跳转到精灵编辑页面

## 潜在依赖和考虑因素

1. **数据传递**：确保帧数据能够正确从帧提取页面传递到精灵编辑页面
2. **用户体验**：确保跳转过程顺畅，没有明显的延迟
3. **错误处理**：添加适当的错误处理，确保在图片加载失败时显示合理的提示

## 风险处理

1. **数据丢失风险**：确保在切换页面时不会丢失用户的工作
2. **兼容性风险**：确保修改后的代码与现有功能兼容
3. **性能风险**：确保处理大量帧时不会影响性能

## 实现细节

1. **代码修改**：在 `sendToFrameEdit` 函数中添加 `img.src = dataUrl` 语句
2. **测试验证**：确保修改后，选择"发送到帧编辑"选项并点击确定后能够自动跳转到精灵编辑页面

## 预期效果

修改后，当用户在导出选项模态框中选择"发送到帧编辑"并点击"确定"按钮时：
1. 系统会将提取的帧转换为精灵编辑页面可识别的格式
2. 创建一个包含所有帧的canvas
3. 将canvas转换为data URL
4. 加载这个data URL作为新的图片
5. 跳转到精灵编辑页面并显示这些帧

这样，发送到帧编辑功能就可以正常工作了。
